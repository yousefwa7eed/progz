# مشروع نظام الماجد الإداري المتكامل
## Almajid Integrated Management System

---

## 1. رؤية المشروع
بناء نظام ERP متكامل لإدارة مؤسسة الماجد، يغطي جميع العمليات التشغيلية والإدارية والمالية، مع التركيز على:
- **الشفافية الكاملة** في جميع العمليات
- **الكفاءة التشغيلية** وتقليل الجهد اليدوي
- **الدقة المالية** والتقارير الموثوقة
- **قابلية التوسع** لسنوات طويلة
- **مجانية** النظام完全 (FOSS)

---

## 2. أهداف الجمعية
| الهدف | الوصف |
|-------|-------|
| تقديم المساعدات | الوصول إلى أكبر عدد من المستفيدين بكرامة |
| تنمية الموارد | بناء قاعدة متبرعين مستدامة |
| الشفافية | كشف كامل للمتبرعين عن أثر تبرعاتهم |
| التميز التشغيلي | أتمتة العمليات وتقليل الهدر |
| الامتثال الشرعي والقانوني | الالتزام بأنظمة الزكاة والرقابة المالية |

## 4. التقنيات المقترحة

### Backend (مجاني بالكامل)
| التقنية | الغرض | البديل |
|---------|-------|--------|
| **Python 3.12 + Django 5.x** | إطار العمل الرئيسي | - |
| **Django REST Framework** | بناء API | - |
| **PostgreSQL 16** | قاعدة البيانات | - |
| **Redis** | تخزين مؤقت (Cache) | - |
| **Celery** | المهام الخلفية (تقارير - إشعارات) | Django Q |
| **Django Allauth** | المصادقة والتسجيل | - |
| **Django Auditlog** | سجل التدقيق | - |
| **django-simple-history** | تتبع التعديلات | - |
| **WeasyPrint / ReportLab** | إنشاء التقارير PDF | - |

### Frontend (مجاني بالكامل)
| التقنية | الغرض |
|---------|-------|
| **Bootstrap 5 + RTL** | واجهة المستخدم (دعم عربي) |
| **HTMX** | تفاعل سريع بدون JavaScript معقد |
| **Chart.js** | الرسوم البيانية |
| **DataTables** | جداول متقدمة مع بحث وفرز |
| **Select2** | قوائم منسدة محسنة |

### البنية التحتية
| المكون | الغرض |
|--------|-------|
| **Linux (Ubuntu 24.04 LTS)** | نظام التشغيل |
| **Nginx** | خادم ويب (Reverse Proxy) |
| **Gunicorn** | تشغيل Django |
| **PostgreSQL** | قاعدة البيانات |
| **Supervisor** | إدارة العمليات |
| **Let's Encrypt (Certbot)** | شهادات SSL مجانية |
| **Restic / Borg** | النسخ الاحتياطي |
| **Git** | إدارة الإصدارات |

### لماذا Django؟
1. **مجاني ومفتوح المصدر** بالكامل
2. **نظام صلاحيات مدمج** (auth + permissions)
3. **Admin Panel جاهز** للمستخدمين الداخليين
4. **ORM قوي** يتعامل مع قواعد البيانات المعقدة
5. **أمان عالي** حماية من SQL Injection, XSS, CSRF
6. **مجتمع كبير** وآلاف الحزم الجاهزة
7. **يدعم RTL** بشكل ممتاز

## 10. المخاطر المحتملة ومعالجتها

| المخاطرة | التأثير | خطة المعالجة |
|----------|---------|--------------|
| ضغط الوقت (شهر) | إطلاق غير مكتمل | تحديد MVP دقيق، البناء المتدرج |

| انقطاع الكهرباء/الإنترنت | فقدان البيانات | نسخ احتياطي محلي + سحابي |
| عدم تبني النظام | فشل المشروع | واجهة بسيطة + تدريب + دعم فني |

## 11. افتراضات التصميم

1. النظام سيتم استضافته **self-hosted** على سيرفر مخصص للجمعية
2. جميع التقنيات **مفتوحة المصدر ومجانية**
3. اللغة الأساسية للواجهة: **العربية** (RTL)
4. العملة الأساسية: **ريال سعودي** (SAR) - قابلة للتغيير
5. السنة المالية: **هجرية + ميلادية**
6. النظام يدعم **التوسع لعدة فروع** في المستقبل
7. التكامل مع **بوابات دفع إلكترونية** مستقبلًا
8. النظام يعمل **24/7** مع نسخ احتياطي يومي
