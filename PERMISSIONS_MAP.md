## 2. نموذج الصلاحيات (Permission Model)

### تسمية الصلاحيات (Naming Convention):
```
{module}.{action}_{entity}
```
مثال: `finance.view_financialentry` , `beneficiaries.add_beneficiary`

### الإجراءات الأساسية لكل كيان:
| الإجراء | الكود | الوصف |
|---------|-------|-------|
| عرض | `view` | مشاهدة البيانات |
| إضافة | `add` | إنشاء سجل جديد |
| تعديل | `change` | تعديل البيانات |
| حذف | `delete` | حذف سجل |
| اعتماد | `approve` | اعتماد معاملة |
| تصدير | `export` | تصدير تقرير |
| طباعة | `print` | طباعة المستندات |

## 4. سياسات الأمان (Security Policies)

### 4.1 كلمات المرور
- الحد الأدنى للطول: 8 أحرف
- يجب أن تحتوي على: حرف كبير، حرف صغير، رقم، رمز خاص
- تغيير كلمة المرور كل 90 يوماً
- لا يمكن تكرار آخر 5 كلمات مرور

### 4.2 الجلسات (Sessions)
- انتهاء الجلسة بعد 30 دقيقة من inactivity
- جلسة واحدة لكل مستخدم (اختياري)
- تسجيل الخروج التلقائي عند انتهاء الجلسة

### 4.3 التحقق الثنائي (2FA)
- إجباري لـ: رئيس مجلس، مدير تنفيذي، مدير مالي، مراجع
- اختياري لبقية المستخدمين
- OTP عبر تطبيق Google Authenticator

### 4.4 IP والموقع
- تسجيل IP لكل جلسة
- تنبيه عند تسجيل الدخول من IP جديد
- حظر IP بعد 5 محاولات فاشلة (لمدة 30 دقيقة)

### 4.5 حماية البيانات
- تشفير: `national_id`, `phone`, `email` في قاعدة البيانات
- إخفاء (Masking): `national_id` في واجهات المستخدم غير المصرح لها
- عدم عرض: `otp_secret` لأي مستخدم

### 4.6 صلاحية الوصول للبيانات (Data Scoping)
- كل مستخدم يرى فقط بيانات **الفرع** الذي ينتمي إليه
- رؤساء الأقسام يرون بيانات قسمهم
- المدير التنفيذي يرى كل شيء
- الاستثناء: التقارير الموحدة للرئيس ومجلس الإدارة
